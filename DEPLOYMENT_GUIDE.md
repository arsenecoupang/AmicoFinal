# AMICO 배포 및 데이터 수집 가이드 📊

## 배포 전략: 30명 학생 사용자 데이터 수집

### 1. 배포 플랫폼: **Netlify** 🌐
- **URL**: [netlify.com](https://netlify.com)
- **이유**: 30명 동시 사용자에게 최적화된 무료 CDN
- **배포 방법**: GitHub 연동으로 자동 배포

#### 설정 단계:
1. Netlify 계정 생성 (GitHub 연동)
2. Repository 연결
3. Build settings: 
   - Build command: `npm run build`
   - Publish directory: `build`
4. 환경변수 설정 (Supabase keys)

### 2. 사용자 데이터 수집 도구들

#### 🔍 **Analytics**: SimpleAnalytics 
- **학생 혜택**: 무료 1년 (월 100K 페이지뷰)
- **수집 데이터**:
  - 일일/주간/월간 활성 사용자
  - 페이지별 방문율
  - 세션 시간
  - 디바이스/브라우저 분포
- **설치**: Script 태그 추가 필요

#### 🚨 **Error Tracking**: Sentry
- **학생 혜택**: 50K 에러, 100K 트랜잭션
- **수집 데이터**:
  - JavaScript 에러
  - 성능 이슈
  - 사용자 세션 리플레이 (500개)
  - 에러 발생 시점의 사용자 행동
- **설치**: React SDK 설치

#### ⚡ **Performance**: New Relic
- **학생 혜택**: 무료 전체 기능 ($300/월 가치)
- **수집 데이터**:
  - 페이지 로딩 시간
  - API 응답 시간  
  - 사용자 상호작용 지연
  - 실시간 트래픽 패턴
- **설치**: Browser agent 설치

#### 🖥️ **Infrastructure**: Datadog
- **학생 혜택**: 2년 무료 Pro 계정
- **수집 데이터**:
  - 서버 리소스 사용량
  - 네트워크 트래픽
  - 데이터베이스 쿼리 성능
  - 사용자 지역별 분포

### 3. 수집할 핵심 데이터

#### 학습 목적 데이터:

- **기능별 인기도**: 온도 시스템, 투표, 채팅 사용률
- **문제점**: 에러 발생 빈도, 사용자 피드백
