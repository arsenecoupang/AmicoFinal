# AMICO 프로젝트 - 한국디지털미디어고등학교 특별전형 실적문

## 프로젝트 개요

AMICO는 A-class Middle Interactive Community Online의 줄임말로, 중학교 3학년 학급 커뮤니티를 위한 실시간 웹 애플리케이션입니다. 이 프로젝트는 React, TypeScript, Supabase를 기반으로 한 현대적인 기술 스택을 활용하여 개발되었으며, 총 8개 반 30명 이상의 학생들이 실제로 사용할 수 있는 완전한 웹 서비스입니다.

개발 기간은 2024년 8월부터 2025년 9월까지 약 1년간 진행되었으며, 현재 https://amicofinal.netlify.app 주소에서 실제 서비스 중입니다. 웹사이트의 탭 이름은 "Amico"로 설정되어 있고, 1반부터 8반까지 총 8개 반을 지원합니다. Supabase 프로젝트 ID는 dajdwwsnhtxruxrwobcq로, 현재 활성 상태를 유지하고 있습니다.

## 사용 기술 및 아키텍처

프론트엔드는 React 19.1.1과 TypeScript를 기반으로 구현되었습니다. React의 컴포넌트 기반 아키텍처를 활용하여 재사용 가능한 UI 구조를 설계했으며, TypeScript를 통해 개발 단계에서 타입 오류를 사전에 방지할 수 있었습니다. 스타일링은 Styled-Components를 사용하여 CSS-in-JS 방식으로 구현했고, 동적 스타일링과 테마 시스템을 통해 일관된 디자인을 유지했습니다. React Router Dom을 통해 SPA 환경에서 클라이언트 사이드 라우팅을 처리했고, Context API를 활용하여 사용자 인증 상태를 전역적으로 관리했습니다.

백엔드는 서버리스 아키텍처를 채택하여 Supabase를 중심으로 구성했습니다. Supabase는 PostgreSQL 기반의 관계형 데이터베이스를 제공하며, 정규화된 테이블 구조를 통해 데이터 무결성을 보장합니다. Supabase Auth를 통해 OAuth 2.0 기반의 이메일 인증 시스템을 구현했고, Supabase Realtime을 활용하여 WebSocket 연결을 통한 실시간 데이터 동기화를 실현했습니다.

배포와 모니터링 측면에서는 Netlify를 통해 JAMstack 아키텍처로 정적 사이트 호스팅을 구현했습니다. GitHub과 연동된 CI/CD 파이프라인을 통해 코드 변경 시 자동으로 빌드와 배포가 이루어집니다. Sentry를 통해 실시간 에러 추적과 성능 모니터링을 구축했고, SimpleAnalytics를 통해 GDPR을 준수하는 사용자 행동 분석 시스템을 도입했습니다.

## 주요 구현 기능

실시간 채팅 시스템은 이 프로젝트의 핵심 기능 중 하나입니다. src/screens/Chat.tsx 파일에서 구현된 이 시스템은 PostgreSQL Trigger와 WebSocket을 결합하여 새로운 메시지가 데이터베이스에 삽입될 때마다 자동으로 모든 클라이언트에 실시간으로 전송됩니다. 무한 스크롤 기능을 통해 채팅 히스토리를 효율적으로 로드하며, 새 메시지 도착 시 화면 하단으로 자동 스크롤되는 UX 최적화를 구현했습니다. React의 useEffect 훅을 활용하여 컴포넌트 마운트 시 Supabase 채널을 구독하고, 언마운트 시 구독을 해제하는 메모리 누수 방지 로직도 포함되어 있습니다.

MVP 투표 시스템은 src/screens/MvpVote.tsx에서 구현되었으며, 민주적 의사결정 도구로 활용됩니다. UUID 기반의 투표 추적 시스템을 통해 중복 투표를 방지하고 데이터 무결성을 보장합니다. PostgreSQL의 집계 함수를 활용하여 실시간으로 순위를 계산하며, 특정 사용자에게는 관리자 권한을 부여하여 투표 초기화 기능을 제공합니다. 투표 테이블은 voter_id, candidate_id, room_id를 외래키로 갖는 정규화된 구조로 설계되어 있으며, 하루 종료 시 MVP 확정 후 자동으로 다음 단계로 진행되는 비즈니스 로직이 구현되어 있습니다.

사용자 인증 및 권한 관리 시스템은 여러 파일에 걸쳐 구현되어 있습니다. src/screens/Login.tsx에서는 회원가입 시 이메일, 별명, 실명의 중복을 검증하는 로직이 구현되어 있습니다. 특히 같은 반 내에서 동일한 실명 사용을 제한하여 실제 학급 환경을 반영했습니다. src/AuthContext.tsx에서는 역할 기반 접근 제어(RBAC)를 구현하여 일반 사용자와 관리자를 구분합니다. scripts/createAdmin.js를 통해 관리자 계정을 자동으로 생성할 수 있으며, 이메일은 admin@amico.dev, 비밀번호는 AmicoDev2025!, 별명은 AmicoDev로 설정됩니다.

자동화된 일일 초기화 시스템은 scripts/dailyReset.ts와 scripts/dailyReset.cjs에 구현되어 있습니다. OpenAI GPT-3.5-turbo 모델을 활용하여 중학생들이 좋아할 만한 밸런스 게임 질문을 자동으로 생성합니다. 환경변수를 통해 개발과 프로덕션 환경을 분리했으며, API 호출 실패 시에는 미리 준비된 질문 풀에서 랜덤으로 선택하는 에러 핸들링 로직이 포함되어 있습니다. package.json의 scripts 섹션에 정의된 npm run daily-reset 명령어를 통해 실행할 수 있습니다.

반응형 UI/UX 설계는 src/components/ 폴더의 여러 파일에서 구현되었습니다. GlobalStyle.ts에서는 전역 CSS 스타일을, theme.ts에서는 색상 시스템과 디자인 토큰을, Header.tsx에서는 네비게이션과 드롭다운 메뉴를, Layout.tsx에서는 전체적인 페이지 레이아웃을 담당합니다. Mobile-First 접근법을 통해 모바일 환경을 우선으로 한 반응형 디자인을 구현했으며, 컴포넌트 재사용성을 높이기 위해 일관된 디자인 토큰을 활용했습니다. 키보드 네비게이션과 시각적 피드백을 제공하여 접근성도 고려했습니다.

## 데이터베이스 설계

데이터베이스는 src/db.ts에서 연결을 관리하고, supabase_setup.sql과 supabase_realtime_setup.sql에서 스키마와 실시간 설정을 정의했습니다. ERD 구조는 정규화 원칙을 엄격히 준수하여 설계되었습니다. profiles 테이블은 사용자 정보를 담고 있으며 id, username, email, realname, class, temperature, role 필드로 구성됩니다. chats 테이블은 채팅 메시지를 저장하며 user_id와 room_id를 외래키로 참조합니다. votes 테이블은 투표 기록을 저장하며 voter_id, candidate_id, room_id를 통해 다대다 관계를 해결합니다. rooms 테이블은 채팅방 정보를 독립적으로 관리합니다.

제1정규형에서는 모든 속성이 원자값으로 구성되도록 했고, 제2정규형에서는 기본키가 아닌 속성들이 기본키에 완전 함수 종속되도록 설계했습니다. 제3정규형에서는 이행적 종속성을 제거하여 데이터 중복을 최소화했습니다. 이러한 정규화를 통해 데이터 무결성을 보장하고 저장 공간을 효율적으로 활용할 수 있었습니다.

## 배포 및 DevOps

배포 설정은 netlify.toml에서 관리되며, .env.example에서 환경변수 템플릿을, package.json에서 빌드 스크립트를, src/index.tsx에서 Sentry 초기화를, DEPLOYMENT_GUIDE.md에서 배포 가이드를 각각 담당합니다. CI/CD 파이프라인은 GitHub Actions를 통해 코드 푸시 시 자동으로 빌드와 테스트가 실행되며, Netlify 배포를 통해 JAMstack 아키텍처의 장점인 글로벌 CDN을 활용합니다. 환경변수 관리를 통해 민감한 API 키들을 안전하게 보호합니다.

모니터링 시스템은 프로덕션 환경에서 발생하는 오류를 즉시 감지하고, 페이지 로드 시간과 API 응답 시간 등의 성능 메트릭스를 측정합니다. 페이지 뷰, 세션 시간 등 사용자 행동 패턴도 분석하여 서비스 개선에 활용합니다. 빌드 시간은 평균 42초이며, 압축된 번들 크기는 약 2MB, First Contentful Paint는 1.5초 미만, WebSocket 기반 실시간 지연은 100ms 미만을 달성했습니다.

## 학습 성과 및 의미

이 프로젝트를 통해 현대 웹 개발의 전반적인 과정을 경험할 수 있었습니다. 단순히 기술을 사용하는 것을 넘어서 각 기술이 해결하는 문제와 그 이유를 깊이 이해하게 되었습니다. 특히 실시간 통신, 데이터베이스 설계, 사용자 인증, 자동화 등 현대 웹 개발의 핵심 기술들을 실무 수준에서 구현해볼 수 있었습니다.

실제 사용자가 있는 서비스를 구축하면서 사용자 중심의 설계 사고를 기를 수 있었고, 중학생이라는 명확한 타겟 사용자를 고려한 직관적인 인터페이스와 접근성을 구현했습니다. 또한 UUID 타입 오류, 환경변수 관리, 실시간 동기화 등 실제 개발 과정에서 발생하는 다양한 문제들을 해결하면서 문제 해결 능력과 디버깅 스킬을 크게 향상시킬 수 있었습니다.

이 프로젝트는 단순한 개발 연습이 아닌, 실제 사용자의 필요를 해결하는 의미 있는 서비스를 만들어본 경험입니다. 30명 이상의 학급 구성원이 실제로 사용할 수 있는 완전한 웹 애플리케이션을 구축함으로써, 웹 개발자로서의 기본기와 실무 능력을 충분히 갖추었다고 자신합니다.
