**배포 주소**: https://amicofinal.netlify.app  
**개발 기간**: 2024년 8월 ~ 2025년 9월 (약 1년)  
**목표 사용자**: 30+ 학급 구성원

## 🛠️ 핵심 기술 스택 및 설계 원리

### **프론트엔드 (클라이언트 사이드)**

- **React 19.1.1 + TypeScript**: 컴포넌트 기반 아키텍처로 재사용 가능한 UI 구조 설계
- **Styled-Components**: CSS-in-JS 방식으로 동적 스타일링 및 테마 시스템 구현
- **React Router Dom**: SPA(Single Page Application)에서 클라이언트 사이드 라우팅 처리
- **Context API**: 전역 상태 관리로 사용자 인증 상태를 컴포넌트 간 공유

### **백엔드 (서버리스 아키텍처)**

- **Supabase PostgreSQL**: 관계형 데이터베이스로 정규화된 테이블 구조 설계
- **Supabase Auth**: OAuth 2.0 기반 이메일 인증 시스템 구현
- **Supabase Realtime**: WebSocket 연결을 통한 실시간 데이터 동기화

### **배포 및 모니터링**

- **Netlify**: JAMstack 아키텍처로 정적 사이트 호스팅 및 CI/CD 파이프라인
- **Sentry**: 실시간 에러 추적 및 성능 모니터링
- **SimpleAnalytics**: 사용자 행동 분석 (GDPR 준수)

### **1. 실시간 채팅 시스템**

**📁 파일 위치**: `src/screens/Chat.tsx`

**기술적 특징**:

- **PostgreSQL Trigger + WebSocket**: 새로운 메시지가 데이터베이스에 삽입될 때마다 자동으로 모든 클라이언트에 실시간 전송
- **무한 스크롤**: 채팅 히스토리를 효율적으로 로드하기 위한 페이지네이션 구현
- **자동 스크롤**: 새 메시지 도착 시 화면 하단으로 자동 이동하는 UX 최적화

**구현 로직** (`src/screens/Chat.tsx` 177-190번째 줄):

```typescript
// 실시간 메시지 구독 (Supabase Realtime)
useEffect(() => {
      ...
      (payload) => {
        const msg = payload.new;
        if (msg.room_id !== roomId) return; // 현재 방만 필터링

        setMsgs((prev) => {
          // 중복 메시지 방지
          const exists = prev.find((m) => m.id === msg.id);
          if (exists) return prev;

          if (tempIdx !== -1) {
            const newMsgs = [...prev];
            newMsgs[tempIdx] = {
              id: msg.id,
              sender: msg.username,
              text: msg.message,
              ts: new Date(msg.created_at).getTime(),
            };
            return newMsgs;
          }

          // 새 메시지 추가
          return [
            ...prev,
            {
              id: msg.id,
              sender: msg.username,
              text: msg.message,
              ts: new Date(msg.created_at).getTime(),
            },
          ];
        });
      }
    )
    .subscribe();

  return () => {
    supabase.removeChannel(channel);
  };
}, [roomId]);
```

```sql
-- 실제 votes 테이블 구조 (supabase_setup.sql)
CREATE TABLE IF NOT EXISTS public.votes (
    id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
    voter_username text NOT NULL,
    voted_username text NOT NULL,
    vote_date date DEFAULT CURRENT_DATE,
    created_at timestamp with time zone DEFAULT now(),
    UNIQUE(voter_username, vote_date)
);

-- profiles 테이블 (사용자 정보)
CREATE TABLE IF NOT EXISTS public.profiles (
    id uuid REFERENCES auth.users(id) ON DELETE CASCADE PRIMARY KEY,
    email text UNIQUE NOT NULL,
    username text UNIQUE NOT NULL,
    realname text NOT NULL,
    temperature integer DEFAULT 0,
    created_at timestamp with time zone DEFAULT now(),
    updated_at timestamp with time zone DEFAULT now()
);
```

## 📊 데이터베이스 설계 및 정규화

**📁 파일 위치**:

- 데이터베이스 연결: `src/db.ts`
- SQL 스키마: `supabase_setup.sql`
- 실시간 설정: `supabase_realtime_setup.sql`

### **ERD(Entity Relationship Diagram) 구조**

```sql
-- 사용자 프로필 (제1정규형 준수)
profiles: id(PK), username, email, realname, class, temperature, role

-- 채팅 메시지 (외래키 제약조건)
chats: id(PK), user_id(FK), room_id(FK), message, created_at

-- 투표 기록 (다대다 관계 해결)
votes: id(PK), voter_id(FK), candidate_id(FK), room_id(FK), created_at

-- 채팅방 (독립 엔티티)
rooms: id(PK), name, created_at, is_active
```

**정규화 원칙 적용**:

- **제1정규형**: 모든 속성이 원자값(Atomic Value)으로 구성
- **제2정규형**: 기본키가 아닌 속성들이 기본키에 완전 함수 종속
- **제3정규형**: 이행적 종속성 제거로 데이터 중복 최소화

useEffect(() => {
  if (!roomId) return;

  const channel = supabase
    .channel("public:chats")
    .on(
      "postgres_changes",
      {
        event: "INSERT",
        schema: "public",
        table: "chats",
      },
      (payload) => {
        const msg = payload.new;
        if (msg.room_id !== roomId) return; // 현재 방만 필터링

        setMsgs((prev) => {
          // 중복 메시지 방지
          const exists = prev.find((m) => m.id === msg.id);
          if (exists) return prev;

          // 임시 메시지를 실제 메시지로 교체
          const tempIdx = prev.findIndex(
            (m) =>
              m.id.startsWith("temp-") &&
              m.sender === msg.username &&
              m.text === msg.message
          );

          if (tempIdx !== -1) {
            const newMsgs = [...prev];
            newMsgs[tempIdx] = {
              id: msg.id,
              sender: msg.username,
              text: msg.message,
              ts: new Date(msg.created_at).getTime(),
            };
            return newMsgs;
          }

          return [
            ...prev,
            {
              id: msg.id,
              sender: msg.username,
              text: msg.message,
              ts: new Date(msg.created_at).getTime(),
            },
          ];
        });
      }
    )
    .subscribe();

  return () => supabase.removeChannel(channel);
}, [roomId]);
